================================================================================
LEAFLORE -- THE COMPLETE JOURNEY & DEVELOPER GUIDE
================================================================================

Welcome to the beginner-friendly guide to LeafLore! This document traces the 
entire evolution of the LeafLore project from a basic Flask application to a 
feature-rich, production-grade, gamified Progressive Web App (PWA).

--------------------------------------------------------------------------------
PART 1: WHAT IS LEAFLORE?
--------------------------------------------------------------------------------
LeafLore is a gamified reading habit tracker designed to help users build 
consistent reading routines. Instead of just showing numbers and progress bars, 
LeafLore turns your reading journey into a Virtual Greenhouse Garden. As you 
read more pages, maintain streaks, and complete books, your garden dynamically 
grows sprouts, trees, and flowers.

--------------------------------------------------------------------------------
PART 2: CHRONOLOGICAL BUILD HISTORY (HOW WE BILT IT)
--------------------------------------------------------------------------------

MILESTONE 1: THE FOUNDATION (Setting Up the Tracker)
--------------------------------------------------
We started with a clean Python web app structure using the Flask micro-framework.
1. Database Schema: Designed database models using SQLAlchemy (SQLite initially)
   to keep track of:
   * Users: Secure passwords hashed via werkzeug.security.
   * Books: Shelf management (Want to Read, Reading, Completed).
   * Reading Logs: Daily page logs (date, pages read, and timed sessions).
   * Chapter Notes: Journal entries linked to specific books.
2. User Authentication: Built register, login, and logout pages secured with 
   CSRF tokens (Flask-WTF) and managed sessions with Flask-Login.
3. Streak Calculator: Programmed a daily streak algorithm that counts 
   consecutive days of reading. If a user skips a day, the streak resets.

MILESTONE 2: INTELLIGENT ANALYTICS (The ML Module)
--------------------------------------------------
To provide real value beyond a simple spreadsheet, we introduced predictions:
1. Reading Pace: Calculates average pages read per day from historical logs.
2. Predictive Analytics: Uses a Machine Learning Logistic Regression model 
   (via scikit-learn) to predict the probability of completing a book on time.
3. Graceful Fallback: If a user is new or has sparse data (cold-start), the app 
   automatically falls back to an intelligent heuristic algorithm based on page 
   progress, notes taken, and focused reading session counts.

MILESTONE 3: THE VIRTUAL GREENHOUSE (Gamification)
--------------------------------------------------
To make tracking fun, we implemented an interactive Greenhouse Garden Canvas 
using HTML5 Canvas and JavaScript.
1. Dynamic Rendering: A JavaScript engine renders SVG-based trees, leafy 
   bushes, and ground sprouts dynamically based on your stats.
   * Taller Plants: Grown by having a higher average page count.
   * More Plants: Unlocked by logging reading sessions on more calendar days.
   * Glow & Particles: A 5+ day streak triggers floating pollen particles 
     and a warm golden sunset gradient.
2. Unlockable Seeds: Earning achievements (like reading 5 books) unlocks rare 
   seed themes (Forest, Meadow, Autumn, Library, Zen, Royal) that users can 
   switch between.
3. Accessibility: Added Calm Mode support. When a user turns on reduced 
   motion, the canvas stops animations (wind sway, particle movement) to save 
   device CPU and provide a comfortable view.
4. Focus Oasis Widget: Integrated a customizable Pomodoro timer and an 
   interactive ambient soundscapes synthesizer mixer (rain, wind, campfire) 
   on the main dashboard to block out external noise while reading.
5. 3D Book Spine Shelf: A visual bookshelf layout utilizing CSS 3D perspectives 
   to render completed book spines with realistic thicknesses, gold foil, and colors.

MILESTONE 4: PROGRESSIVE WEB APP & OFFLINE SYNC
--------------------------------------------------
We converted LeafLore into a fully installable Progressive Web App (PWA):
1. Installability: Added a manifest.json file and a service worker (sw.js). 
   Users on mobile get "Add to Home Screen" prompts, while desktop users get 
   a polished slide-in install banner.
2. Local Sync Queue: If the user loses internet connectivity, the application 
   displays an amber warning banner. Quick logs and focus sessions are stored 
   in a local browser database called IndexedDB.
3. Auto-Replay: When the connection is restored, a script replays the queued 
   reading logs to the /dashboard/api/offline-sync endpoint, merging them.
4. Vocabulary Word Bank: Added a personal dictionary card library to collect 
   unfamiliar words, definitions, and text contexts.
5. 3D Study Flashcards: Study collected vocabulary words with a 3D-flipping 
   spaced repetition review interface.

MILESTONE 5: ZERO-COST PRODUCTION HARDENING
--------------------------------------------------
Finally, we prepared LeafLore for deployment to cloud platforms at zero cost:
1. WSGI Server: Swapped the Flask developer server with Gunicorn configured 
   in a Procfile.
2. Database Migrations: Integrated Flask-Migrate so you can modify database 
   columns in the future without destroying user data.
3. Security Redirection: Configured Flask-Talisman to force HTTPS, protect 
   against Clickjacking (X-Frame-Options), and block cross-site scripting (CSP).
4. Rate Limiting: Added Flask-Limiter to register/login routes to prevent 
   automated brute-force attacks.
5. Asset Serving: Integrated WhiteNoise so your CSS, JavaScript, and icons are 
   automatically compressed (Gzip/Brotli) and cached directly by Gunicorn.

MILESTONE 6: PAGES DENSITY REDESIGN (INSIGHTS & SETTINGS), DATA RESETTING, & ADMIN CONTROLS
--------------------------------------------------
In this phase, we optimized the information architecture, enhanced data portability, and introduced role-based administrative controls:
1. Information Architecture (IA) Redesign:
   * Dedicated Insights View: Moved advanced reading trends (monthly graphs, reading speeds, and ML recommendations) from the home page to a dedicated `/insights` tab (`insights.html`) to simplify page density.
   * Dedicated Settings View: Centralized preferences, daily page targets, Calm Mode controls, and account deletion forms to a dedicated `/settings` page (`settings.html`).
2. Data Portability & Exports: Integrated downloading backups directly from the Settings page:
   * CSV download of all reading logs.
   * Markdown download of all chapter reflections and notes.
   * JSON archive download of full database records.
3. Account Data Reset (Blank Slate): Added a warning card in Settings allowing users to wipe all books, logs, streaks, and achievements to start fresh with a blank slate, without deleting their user login credentials.
4. Dedicated Admin Dashboard: Added a `/admin` route blueprint guarded by `@admin_required`. It combines system specs and database diagnostics (Ops Health) with an interactive User Management panel showing:
   * User impersonation session logins.
   * Remote-seeding of demo logs and data for any user account.
   * Resets of reading history to a blank slate for any user.
   * Role status toggles (`is_admin`) to delegate dashboard control.
   * Permanent account deletions for any user.
5. Navigation & UI: Displays an "Admin" navbar shortcut and an "Admin Console" profile dropdown link only when `current_user.is_admin` is True.
6. Guided Onboarding Tour: Integrated a 6-slide welcome tour modal that appears automatically on a user's first login to guide them through all features (Welcome, Focus Oasis, Reading Garden garden-grow mechanics, AI & predictive models, achievements seeds, and PWA offline sync exports).
7. Yearly Reading Wrap-up: Added a dedicated `/wrap` page highlighting the user's reading year, showing page milestones, streak records, and favorite reading genres.

PART 3: RUNNING THE PROJECT LOCALLY (FOR BEGINNERS)
--------------------------------------------------------------------------------

Here is how you can spin up the project on your own machine:

1. PREREQUISITE CHECKLIST
-------------------------
* Install Python 3.11 or 3.12 on your machine.
* Install a code editor (like VS Code).

2. STEP-BY-STEP INSTALLATION
----------------------------
Open your terminal (Command Prompt, PowerShell, or Bash) in the project folder:

  # 1. Create a virtual environment (keeps dependencies isolated)
  python -m venv venv

  # 2. Activate the virtual environment
  # On Windows:
  venv\Scripts\activate
  # On Mac/Linux:
  source venv/bin/activate

  # 3. Install all pinned libraries
  pip install -r requirements.txt

  # 4. Set environment variables (for local development)
  # On Windows (PowerShell):
  $env:FLASK_ENV="development"
  $env:FLASK_APP="wsgi.py"
  # On Mac/Linux/Git Bash:
  export FLASK_ENV="development"
  export FLASK_APP="wsgi.py"

  # 5. Initialize the database schema
  flask db upgrade

  # 6. Start the local server
  flask run

Your app is now running at: http://127.0.0.1:5000

3. GENERATE SAMPLE DATA
-----------------------
To see the garden and ML predictions in action immediately:
1. Run `python seed.py` to reset the database and seed default accounts.
2. Log in with the following credentials:
   * **Regular User:** `demo@leaflore.app` / `leaflore123` (username: `demo`)
     - Contains seeded mock books, notes, logs, and a reading streak.
   * **Admin User:** `admin@leaflore.app` / `adminpassword` (username: `admin`)
     - Includes access to the dedicated Admin Dashboard at http://127.0.0.1:5000/admin.
3. Alternatively, register a new account and click "Seed Demo Data" on the dashboard. You can clear this data at any time from the Settings page.

PART 4: DEPLOYING TO THE CLOUD FOR FREE
--------------------------------------------------------------------------------

LeafLore is pre-configured to be hosted on Render.com backed by a free 
PostgreSQL database (e.g., Supabase or Neon.tech).

STEP 1: SET UP YOUR GITHUB REPOSITORY
-------------------------------------
1. Initialize a git repository locally:
   git init
2. Commit your code:
   git add .
   git commit -m "initial commit"
3. Push the code to a repository on GitHub.

STEP 2: CREATE A POSTGRESQL DATABASE
------------------------------------
1. Go to Supabase (https://supabase.com) or Neon (https://neon.tech) and 
   create a free project.
2. Go to the Database Settings and copy your connection string (URI format, 
   starting with postgresql://...).

STEP 3: DEPLOY ON RENDER
------------------------
1. Go to Render.com and create a free account.
2. Click New + -> Blueprints.
3. Connect your GitHub repository. Render will automatically read the 
   render.yaml file in the project.
4. Render will ask you for configuration parameters:
   * DATABASE_URL: Paste the connection string from Supabase/Neon.
   * SECRET_KEY: Leave blank (Render auto-generates a secure key).
   * FLASK_ENV: Set to production.
5. Click Apply. Render will install dependencies, apply database migrations 
   (flask db upgrade), and start Gunicorn.

Once complete, your live reading app URL will be available! You can configure 
a free account on UptimeRobot.com to ping your app's /health endpoint every 
5 minutes to keep it from going to sleep.
